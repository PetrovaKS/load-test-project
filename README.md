# Load Testing Service

## Клонировать репозиторий

git clone https://github.com/PetrovaKS/load-test-project.git
cd load-test-project

## Запустить все сервисы

docker-compose up --build

## Проект будет доступен по адресам:

Frontend: http://localhost:8080
Backend API: http://localhost:3000

## Последовательные оптимизации и их эффективность

### Список примененных оптимизаций:

Пагинация (limit 10) - основная оптимизация
Добавление индексов - дополнительное ускорение
Connection Pooling - управление соединениями
Кэширование - снижение нагрузки на БД
Оптимизация запросов + сжатие - финальная оптимизация

### Результаты улучшений:

Наиболее показательный тест (1000 запросов с задержкой 100мс):

До оптимизаций: 1143,62 сек
После всех оптимизаций: 120,1 сек
Ускорение в 9,5 раз (уменьшение времени на 89%)

Без задержки (1000 запросов):
До: 1014,03 сек
После: 9,2 сек
Ускорение в 110 раз (уменьшение времени на 99%)

### Ключевые выводы:

Пагинация дала наибольший прирост - уменьшение времени с 1000+ сек до ~15 сек
Индексы особенно эффективны при параллельных запросах с задержкой
Кэширование и оптимизация запросов дали дополнительное ускорение 20-30%
